## SDD-RIPER 需求规格说明书：mimi

### 1. [R]equirement (需求定义)

* **核心痛点**：在 LLM 时代，用户在输入 Prompt 后需要经历几秒到十几秒的流式输出等待。这段时间容易导致注意力涣散或产生等待焦虑。
* **目标定位**：一款提供情绪价值的 Mac 桌面陪伴工具，能在微小的碎片时间里充当视觉锚点。
* **核心功能 (MVP)**：
  * **视线追踪**：实时追踪系统鼠标或输入光标的位置。
  * **状态感知**：根据用户的操作频率，自动切换不同的待机/活动动画。
  * **无感共存**：完全不干扰用户的正常工作流。

### 2. [I]nterface (边界与接口规范)

这是系统最核心的交互契约与技术边界。

* **用户交互边界**：
  * **视觉层**：窗口必须处于 `floating` 级别（全局置顶），且背景完全透明。
  * **操作层**：默认状态下开启“事件穿透”（Ignores Mouse Events），用户的点击操作直接作用于底层窗口。应用设置通过 Mac 顶部的系统菜单栏 (Menu Bar) 图标进行管理。
* **系统级 API 依赖**：
  * **鼠标追踪**：依赖 AppKit 的 `NSEvent.addGlobalMonitorForEvents`。
  * **光标追踪**：强制依赖 macOS `Accessibility API`，通过 `AXUIElement` 获取当前焦点，并查询 `kAXBoundsForRangeParameterizedAttribute`。
* **核心降级策略 (Fallback)**：
  * 由于部分非原生应用（如某些 Electron 套壳应用）对 Accessibility 支持不佳，系统必须具备**热切换能力**：当无法获取文本光标位置时，无缝降级为追踪鼠标指针的全局坐标。

### 3. [P]roduct (产品表现与状态机)

定义“狸花猫”的核心行为逻辑（State Machine）。

* **Focus (高频输入态)**：
  * 触发条件：检测到连续的键盘输入或鼠标快速移动。
  * 动画表现：耳朵竖起，眼睛（瞳孔）精准计算角度并死死盯住光标位置，头部跟随微调。
* **Wait (等待/思考态)**：
  * 触发条件：光标停滞或无操作持续 3-10 秒（模拟等待 LLM 响应的场景）。
  * 动画表现：视线焦点放松，猫咪开始“揣手手”、打哈欠，或缓慢甩动尾巴，提供舒缓的情绪反馈。
* **Idle (休眠态)**：
  * 触发条件：无操作超过 10 秒。
  * 动画表现：闭眼睡觉，随着呼吸频率身体微微起伏，偶尔耳朵抖动。

### 4. [E]ngineering (工程架构指南)

* **技术栈选择**：原生 Swift + AppKit（不建议使用跨平台框架，以确保对系统底层 Accessibility API 的最高控制权）。
* **坐标系转换**：Mac 支持复杂的多显示器排列。必须实现一个强健的坐标转换工具类，将屏幕全局坐标系正确映射到宠物窗口的本地坐标系中，并使用三角函数（如 `atan2`）计算瞳孔旋转的准确弧度。
* **性能基线**：作为伴生工具，资源占用是红线。CPU 占用率需严格控制在 1-2% 以下，内存占用极小化，避免成为“电量杀手”。

### 5. [R]elease (发布与验收标准)

* **权限合规**：应用启动时，必须有清晰的弹窗引导用户前往“系统设置 -> 隐私与安全性 -> 辅助功能”中开启权限，否则核心接口将直接失效。
* **MVP 验收标准**：
  * 成功在主屏幕渲染无边框透明猫咪。
  * 在原生应用（如备忘录、Xcode）中成功捕获闪烁的文本光标并完成视线跟随。
  * 在不支持的场景下，成功触发 Fallback 机制追踪鼠标。
